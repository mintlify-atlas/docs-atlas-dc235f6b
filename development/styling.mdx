## CSS Architecture

AnimeAho Client uses a modular CSS architecture with three main layers:

1. **Global Styles** (`pages.css`) - Variables, resets, common components
2. **Page-Specific Styles** (`index.css`, `anime.css`, etc.) - Page layouts
3. **Responsive Styles** (`responsive.css`) - Media queries for all breakpoints

<Note>
  CSS files must be loaded in this exact order for styles to cascade correctly.
</Note>

## Design System

### CSS Variables

All colors, fonts, and spacing are defined as CSS variables in `pages.css`:

```css css/pages.css
:root {
    /* Typography */
    --font-display: 'Exo 2', sans-serif;
    --font-body: 'Nunito', sans-serif;

    /* Colors */
    --color-bg:            #04132e;
    --color-bg-card:       #071e45;
    --color-header:        #071d42;
    --color-footer:        #06296b;
    --color-accent:        #1a6bc4;
    --color-accent-glow:   rgba(26, 107, 196, 0.35);
    --color-accent-alt:    #3392ff;
    --color-text:          #eef2ff;
    --color-text-muted:    #7a9cc4;
    --color-border:        rgba(255, 255, 255, 0.07);
    --color-sidebar-hover: rgba(26, 107, 196, 0.15);
}
```

### Using CSS Variables

Always use CSS variables for consistency:

```css
/* ✅ Good: Uses CSS variables */
.my-component {
    background-color: var(--color-bg-card);
    color: var(--color-text);
    border: 1px solid var(--color-border);
    font-family: var(--font-body);
}

/* ❌ Bad: Hardcoded values */
.my-component {
    background-color: #071e45;
    color: #eef2ff;
    border: 1px solid rgba(255, 255, 255, 0.07);
    font-family: 'Nunito', sans-serif;
}
```

## Component Styling Patterns

### BEM Naming Convention

AnimeAho uses BEM (Block Element Modifier) for CSS class names:

```css
/* Block */
.anime-card { }

/* Element */
.anime-card__image { }
.anime-card__name { }
.anime-card__badge { }

/* Modifier */
.anime-card--episode { }
.anime-card--featured { }
```

## Responsive Design

### Breakpoints

AnimeAho uses these standard breakpoints:

```css
/* Desktop Medium */
@media (max-width: 1400px) { }

/* Desktop Small / Tablet Large */
@media (max-width: 1150px) { }

/* Tablet */
@media (max-width: 900px) { }

/* Mobile Large */
@media (max-width: 640px) { }

/* Mobile Small */
@media (max-width: 521px) { }

/* Mobile Tiny */
@media (max-width: 360px) { }
```

### Responsive Grid Example

```css css/responsive.css
/* Base: Auto-fill responsive grid */
#latest-episodes.main__animes-container {
    grid-template-columns: repeat(auto-fill, minmax(var(--ep-card-width), 1fr));
}

#latest-animes.main__animes-container {
    grid-template-columns: repeat(auto-fill, minmax(var(--card-width), 1fr));
}

/* Tablet: Adjust card sizes */
@media (max-width: 900px) {
    :root {
        --card-width:     145px;
        --ep-card-width:  170px;
        --ep-card-height: 115px;
    }
}

/* Mobile: Fixed 2-column grid */
@media (max-width: 640px) {
    :root {
        --card-width:     130px;
        --ep-card-width:  150px;
        --ep-card-height: 100px;
        --card-gap:       10px;
        --ep-gap:         10px;
    }

    #search-animes.main__animes-container {
        grid-template-columns: repeat(2, 1fr);
    }
}
```

## Styling Best Practices

<AccordionGroup>
  <Accordion title="Use CSS variables for all colors">
    ```css
    /* ✅ Good */
    .component {
        color: var(--color-text);
        background: var(--color-bg-card);
    }

    /* ❌ Bad */
    .component {
        color: #eef2ff;
        background: #071e45;
    }
    ```
  </Accordion>

  <Accordion title="Follow BEM naming convention">
    ```css
    /* ✅ Good: Clear BEM structure */
    .anime-card { }
    .anime-card__image { }
    .anime-card__title { }
    .anime-card--featured { }

    /* ❌ Bad: Generic or nested names */
    .card { }
    .card .image { }
    .featured-card { }
    ```
  </Accordion>

  <Accordion title="Use logical properties when possible">
    ```css
    /* ✅ Good: Logical properties */
    .element {
        padding-inline: 1rem;
        margin-block-end: 2rem;
        inset: 0;
    }

    /* ❌ OK but less flexible */
    .element {
        padding-left: 1rem;
        padding-right: 1rem;
        margin-bottom: 2rem;
        top: 0; right: 0; bottom: 0; left: 0;
    }
    ```
  </Accordion>

  <Accordion title="Optimize transitions and animations">
    ```css
    /* ✅ Good: Smooth performance */
    .element {
        transition: transform 0.25s ease, opacity 0.25s ease;
        will-change: transform;
    }

    /* ❌ Bad: Janky animations */
    .element {
        transition: all 0.25s;
        /* Animates everything including layout properties */
    }
    ```
  </Accordion>
</AccordionGroup>

## Testing Responsive Styles

<Steps>
  <Step title="Open DevTools">
    Press `F12` or right-click and select "Inspect"
  </Step>

  <Step title="Toggle Device Toolbar">
    Press `Ctrl+Shift+M` (Windows/Linux) or `Cmd+Shift+M` (Mac)
  </Step>

  <Step title="Test Breakpoints">
    Test these common viewport widths:
    - 1920px (Desktop HD)
    - 1366px (Laptop)
    - 900px (Tablet)
    - 640px (Mobile Large)
    - 375px (Mobile Small)
  </Step>

  <Step title="Verify Layout">
    Check:
    - Text is readable at all sizes
    - Images scale properly
    - Navigation is accessible
    - No horizontal scroll
    - Touch targets are 44px+ on mobile
  </Step>
</Steps>

## Accessibility

```css
/* Focus visible for keyboard navigation */
*:focus-visible {
    outline: 2px solid var(--color-accent);
    outline-offset: 2px;
}

/* Hide elements visually but keep for screen readers */
.sr-only {
    position: absolute;
    width: 1px;
    height: 1px;
    padding: 0;
    margin: -1px;
    overflow: hidden;
    clip: rect(0, 0, 0, 0);
    white-space: nowrap;
    border-width: 0;
}

/* Ensure sufficient contrast */
.text {
    color: var(--color-text); /* #eef2ff on #04132e = 13.82:1 */
}
```
