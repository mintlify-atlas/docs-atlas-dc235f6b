The AnimeAho Client follows a modular architecture with clear separation of concerns. This guide explains the purpose of each directory and provides examples of the files within them.

## Overview

The project is built with vanilla JavaScript using ES6 modules, with no build step or bundler required. All code is organized into logical directories based on functionality.

```
animeaho-client/
├── index.html              # Main entry point
├── anime.html              # Anime detail page
├── browse.html             # Browse/search page
├── watch.html              # Video player page
├── search.html             # Search results page
├── favorites.html          # User favorites page
├── register.html           # User registration page
├── 404.html                # Error page
├── assets/                 # Static assets
│   ├── icons/              # Favicon and icon files
│   └── images/             # Logo and images
├── css/                    # Stylesheets
│   ├── index.css           # Home page styles
│   ├── anime.css           # Anime detail styles
│   ├── browse.css          # Browse page styles
│   ├── watch.css           # Video player styles
│   ├── search.css          # Search page styles
│   ├── components.css      # Component styles
│   ├── pages.css           # Global page styles
│   ├── auth.css            # Authentication styles
│   └── responsive.css      # Media queries
└── js/                     # JavaScript modules
    ├── components/         # Reusable UI components
    ├── config/             # Configuration files
    ├── models/             # Data models
    ├── pages/              # Page-specific logic
    ├── services/           # API services
    └── utils/              # Utility functions
```

## HTML Pages

The application uses multiple HTML pages for different views:

- **`index.html`** - Home page showing latest episodes and airing anime
- **`anime.html`** - Detailed view of a single anime with episodes
- **`browse.html`** - Browse anime with filters
- **`watch.html`** - Video player for watching episodes
- **`search.html`** - Search results page
- **`favorites.html`** - User's favorite anime collection
- **`register.html`** - User registration and login
- **`404.html`** - Error page for not found resources

<Note>
  Each HTML page imports its corresponding JavaScript module from `js/pages/` using `<script type="module">` tags.
</Note>

**Example Component:** `AnimeCard.js` creates anime cards with cover images and titles:

```javascript js/components/AnimeCard.js:24-63
import { navigateToAnimeDetail, navigateToEpisode } from '../utils/navigation.js';

/**
 * Creates an anime card with cover, title and optional badge.
 * 
 * @param {Object} anime - Anime data { title, cover, slug }
 * @param {string|null} badgeText - Badge text (e.g., "Episode 3")
 * @param {boolean} isDetailView - true = navigate to detail | false = to episode
 * @returns {HTMLElement} Card ready to insert in DOM
 */
export function renderAnimeCard(anime, badgeText = null, isDetailView = true) {
    const article = document.createElement('article');
    article.className = 'anime-card';
    article.setAttribute('data-anime-slug', anime.slug);

    const imageLink = document.createElement('a');
    imageLink.className = 'anime-card__a';
    imageLink.href = '#';

    const image = document.createElement('img');
    image.className = 'anime-card__image';
    image.src = anime.cover;
    image.alt = `Portada de ${anime.title}`;
    image.loading = 'lazy';

    imageLink.appendChild(image);

    if (badgeText) {
        const badge = document.createElement('div');
        badge.className = 'anime-card__badge';
        badge.textContent = badgeText;
        imageLink.appendChild(badge);
    }

    const titleElement = document.createElement('p');
    titleElement.className = 'anime-card__name';
    titleElement.textContent = anime.title;

    article.appendChild(imageLink);
    article.appendChild(titleElement);

    imageLink.addEventListener('click', (event) => {
        event.preventDefault();
        handleCardNavigation(anime.slug, isDetailView);
    });

    return article;
}
```

**Example:** Anime service provides methods to fetch data:

```javascript js/services/animeService.js:15-31
import { ANIME_API_BASE_URL, ANIME_ENDPOINTS } from "../config/api.config.js";
import { AnimeBySlugResponse, ResponseError } from "../models/Anime.js";

/**
 * Gets complete anime information by slug.
 */
export async function getAnimeBySlug(animeSlug) {
    const endpoint = ANIME_API_BASE_URL + ANIME_ENDPOINTS.ANIME_SLUG(animeSlug);
    try {
        const response = await fetch(endpoint);
        const data = await response.json();
        if (!response.ok) return new ResponseError(data);
        if (!data.success) return new ResponseError(data);
        return new AnimeBySlugResponse(data);
    } catch (error) {
        console.error("Error al obtener anime:", error);
        return new ResponseError({...});
    }
}
```

## Module Import Pattern

The project uses ES6 module imports throughout:

```javascript
// Import from services
import { getAnimeBySlug } from '../services/animeService.js';

// Import from components
import { renderAnimeCard } from '../components/AnimeCard.js';

// Import from config
import { ANIME_API_BASE_URL } from '../config/api.config.js';

// Import from models
import { AnimeBySlugResponse } from '../models/Anime.js';

// Import from utils
import { navigateToAnimeDetail } from '../utils/navigation.js';
```

<Warning>
  Always include the `.js` file extension in import statements. ES6 modules in browsers require explicit file extensions.
</Warning>

## Data Flow

Here's how data flows through the application:

<Steps>
  <Step title="Page initialization">
    HTML page loads and executes the corresponding page script from `js/pages/`
  </Step>

  <Step title="Service call">
    Page script calls a service function from `js/services/` to fetch data
  </Step>

  <Step title="API request">
    Service constructs the endpoint URL using `js/config/api.config.js` and makes a fetch request
  </Step>

  <Step title="Data modeling">
    API response is parsed and transformed into a model class from `js/models/`
  </Step>

  <Step title="Component rendering">
    Page script passes the model data to component functions from `js/components/`
  </Step>

  <Step title="DOM update">
    Components create and return HTML elements that are inserted into the DOM
  </Step>
</Steps>

## Best Practices

When working with this structure:

<CardGroup cols={2}>
  <Card title="Keep components pure" icon="cube">
    Components should only handle rendering and user interaction, not data fetching.
  </Card>
  
  <Card title="Use services for API calls" icon="cloud">
    All HTTP requests should go through service modules for consistency.
  </Card>
  
  <Card title="Define data with models" icon="database">
    Always parse API responses through model classes for type safety.
  </Card>
  
  <Card title="Extract common logic" icon="screwdriver-wrench">
    Reusable functionality belongs in `js/utils/` modules.
  </Card>
</CardGroup>
