This guide will help you set up and run the AnimeAho Client application locally. You'll have a working anime streaming platform in just a few minutes.

## Prerequisites

Before you begin, ensure you have:

- A modern web browser (Chrome, Firefox, Safari, or Edge)
- A local web server (Live Server, Python's HTTP server, or Node.js http-server)
- Git installed on your machine

## Setup Instructions

<Steps>
  <Step title="Clone the repository">
    Clone the AnimeAho Client repository to your local machine:

    ```bash
    git clone https://github.com/yourusername/animeaho-client.git
    cd animeaho-client
    ```
  </Step>

  <Step title="Open the project">
    The project is a vanilla JavaScript application with no build step required. Simply navigate to the project directory:

    ```bash
    cd animeaho-client
    ```
  </Step>

  <Step title="Start a local server">
    Since the project uses ES6 modules, you need to serve it through a local web server. Choose one of the following options:

    <CodeGroup>
      ```bash VS Code Live Server
      # Right-click on index.html and select "Open with Live Server"
      # Or install the Live Server extension and click "Go Live"
      ```

      ```bash Python
      python -m http.server 8000
      # Then visit http://localhost:8000
      ```

      ```bash Node.js
      npx http-server -p 8000
      # Then visit http://localhost:8000
      ```
    </CodeGroup>
  </Step>

  <Step title="Access the application">
    Open your browser and navigate to:

    ```
    http://localhost:8000
    ```

    You should see the AnimeAho home page with:
    - Latest episodes in the main content area
    - Currently airing anime in the sidebar
    - A search bar in the header
  </Step>
</Steps>

## Quick Example: Loading Anime Data

The home page automatically loads anime content using the built-in services. Here's how the main page initializes:

```javascript js/pages/index.js
import {
    getAiringAnimes,
    getLatestEpisodes,
    searchAnimeFilter,
} from '../services/animeService.js';
import { renderAnimeCard } from '../components/AnimeCard.js';

async function initializeHomePage() {
    try {
        await loadAiringAnimes();
        await loadLatestEpisodes();
        await loadLatestAnimes();
        addListenerInput();
    } catch (error) {
        console.error('Error al inicializar la página principal:', error);
        displayErrorMessage('Error al cargar el contenido.');
    }
}

// Initialize when DOM is loaded
document.addEventListener('DOMContentLoaded', initializeHomePage);
```

<Note>
  The application fetches anime data from the AnimeFlv public API at `https://animeflv.ahmedrangel.com`. No API key is required for basic functionality.
</Note>

## Backend Integration (Optional)

For advanced features like user authentication, favorites, and comments, you'll need to connect to the AnimeAho backend:

1. The backend API URL is configured in `js/config/api.config.js`:

```javascript js/config/api.config.js
// API backend de AnimeAho (proxy y autenticación)
export const BACKEND_API_BASE_URL = 'https://animeaho-backend.vercel.app/api';

export const BACKEND_ENDPOINTS = {
    // Autenticación
    LOGIN: '/auth/login',
    REGISTER: '/auth/register',
    LOGOUT: '/auth/logout',
    
    // Favoritos
    GET_FAVORITES: '/favorites',
    ADD_FAVORITE: '/favorites',
    
    // Comentarios
    GET_COMMENTS: (animeId, episodeId) => `/comments/${animeId}/${episodeId}`,
    ADD_COMMENT: '/comments',
};
```

2. Update the `BACKEND_API_BASE_URL` if you're running your own backend instance.

## Troubleshooting

<AccordionGroup>
  <Accordion title="CORS errors in the browser console">
    Make sure you're serving the application through a web server, not opening the HTML file directly. ES6 modules require a proper HTTP server.
  </Accordion>

  <Accordion title="Anime images not loading">
    Check your internet connection. The application fetches images from external CDNs. Some images may be blocked by your network or browser extensions.
  </Accordion>

  <Accordion title="API requests failing">
    Verify that the API endpoints in `js/config/api.config.js` are accessible. The public API may occasionally be unavailable.
  </Accordion>
</AccordionGroup>
